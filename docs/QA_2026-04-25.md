## Issues found and fixed

1. **`/icon.svg` 404** — A preload hint added in Ship #6 referenced an asset that didn't exist. Removed the preload and created `app/icon.svg` so Next.js serves it via the icon convention.
2. **`/flatten-pdf` 404** — 4 SEO landings linked to /flatten-pdf in their `related` arrays but the page didn't exist. Added a complete SEO landing entry + app route.
3. **Hostinger LSAPI 503 spike during heavy crawling** — Triggered an LSAPI worker restart via SSH. Reduced sweep parallelism from 30 → 5. Documented as known issue (Task #20).

## Issues NOT fixed (pre-existing, not introduced by this session)

- **Hostinger LSAPI 503 on /sitemap.xml under load** — Still flaky under aggressive concurrent load. Out of scope; tracked as Task #20.
- **`/favicon.ico` 404** — Browsers fall back to the new app/icon.svg. Could ship a literal favicon.ico at any time but not user-facing-critical.

## Conclusion

The site is in a healthy state. Every URL on the sitemap returns 200 with substantive content. Every new schema type (Article, BreadcrumbList, HowTo, FAQPage, CollectionPage) emits correctly.
